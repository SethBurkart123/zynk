# Kitchen Sink Example

This example demonstrates all the features of Zynk:

- **Users Module**: Basic CRUD operations with Pydantic models
- **Weather Module**: Streaming/channel support with real-time updates
- **Tasks Module**: Complex nested models and enums

## Quick Start

### 1. Start the Backend

```bash
cd backend

# Install Zynk (from parent directory)
pip install -e ../../

# Run the server
python main.py
```

You should see:
```
==================================================
  Zynk - Kitchen Sink API
==================================================
  Server:     http://127.0.0.1:8000
  Mode:       Development
  Commands:   16
  TypeScript: ../frontend/src/generated/api.ts
==================================================
```

### 2. Start the Frontend

In a new terminal:

```bash
cd frontend

# Install dependencies
npm install

# Start Vite dev server
npm run dev
```

Open http://localhost:5173

## Features Demonstrated

### Users Tab
- List users
- Create new users
- Delete users
- Search users
- View user details

### Weather Tab
- Fetch current weather
- 7-day forecast
- **Live streaming** with Server-Sent Events

### Tasks Tab
- Task management with priorities
- Labels/tags
- Statistics dashboard
- Status updates

## Project Structure

```
kitchen-sink/
├── backend/
│   ├── main.py       # Bridge setup & entry point
│   ├── users.py      # User management commands
│   ├── weather.py    # Weather commands with streaming
│   └── tasks.py      # Task management with nested models
└── frontend/
    ├── src/
    │   ├── generated/      # Auto-generated by Zynk
    │   │   ├── api.ts      # Main API client
    │   │   └── _internal.ts
    │   ├── components/
    │   │   ├── UsersDemo.tsx
    │   │   ├── WeatherDemo.tsx
    │   │   └── TasksDemo.tsx
    │   ├── App.tsx
    │   └── main.tsx
    └── package.json
```

## Hot Reload Demo

Try editing `backend/users.py`:

1. Add a new field to the `User` model
2. Save the file
3. Watch the Python server reload
4. Watch the TypeScript get regenerated
5. The frontend updates via HMR!
